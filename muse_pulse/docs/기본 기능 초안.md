# # MPP: Muse Pulse Project 할 일

> AI 기반 한국 주식시장 예측 및 자동매매 시스템 개발 계획

---

## 1. 데이터 수집 (Data Ingestion)

기존 가격 파이프라인에 텍스트·이벤트 데이터를 더해 **멀티모달 입력**을 구성한다.

- [ ] 주가/거래 데이터: 기존 `pykrx`, `FinanceDataReader` 파이프라인 확장 (일봉 → 분봉/틱까지 확장 여부 결정)
- [ ] 뉴스 수집: 네이버 금융 뉴스, 한경, 매경, 연합뉴스, 이투데이 등 크롤러 구축 (robots.txt·저작권 확인)
- [ ] 영문 뉴스/글로벌 이슈: Reuters, Bloomberg(유료), Yahoo Finance RSS
- [ ] 공시 데이터: DART Open API (실적, 내부자 거래, 지분변동)
- [ ] 기업 리포트: 증권사 리서치 PDF 크롤링 (한경컨센서스 등)
- [ ] SNS·커뮤니티(선택): 트위터/X API, 종목토론방, 텔레그램 채널
- [ ] 거시 데이터: ECOS(한국은행), FRED, World Bank (기존 파이프라인 재사용)
- [ ] 데이터 레이크 설계: 원천(raw) → 정제(cleaned) → 피처(features) 3계층 저장 (파일시스템 or S3/Parquet)
- [ ] 수집 스케줄러: Airflow 또는 cron 기반 일·시간 단위 배치 정의

---

## 2. 뉴스·텍스트 전처리 (Text Processing)

한국어 금융 텍스트 특유의 어려움을 해결하는 별도 파이프라인.

- [ ] HTML 파싱 및 본문 추출 (newspaper3k, trafilatura)
- [ ] 한국어 토크나이저 선택 (KoNLPy/Mecab 또는 KLUE-BERT tokenizer)
- [ ] 중복·유사 기사 제거 (MinHash, SimHash)
- [ ] 종목 엔티티 인식(NER): 기사 → 관련 종목코드 매핑 (동의어 사전 구축)
- [ ] 이벤트 분류: 실적, M&A, 소송, 경영진 교체, 규제 등 카테고리 태깅
- [ ] 감성 분석 모델: KoFinBERT, KR-FinBert 또는 LLM 기반 제로샷 스코어링
- [ ] 시계열 정렬: 기사 발행 시각 기준으로 익일 시초가 또는 다음 영업일과 매칭 (look-ahead bias 방지)
- [ ] 텍스트 피처화: 감성 점수, 주제 분포, 언급 빈도, 뉴스 볼륨 변화율 등

---

## 3. 피처 통합 및 라벨링 (Feature Fusion & Labeling)

기존 48개 기술지표 + 텍스트 피처 + 거시 피처를 **하나의 입력 텐서**로 통합한다.

- [ ] 시점 정렬(point-in-time correctness): 뉴스 피처는 해당 시점 이전 정보만 사용
- [ ] 피처 정규화: 기존 Z-score 방식 유지, 텍스트 피처는 robust scaler 고려
- [ ] 피처 중요도 1차 분석: SHAP, permutation importance로 불필요 피처 가지치기
- [ ] 라벨링 전략 확정: 기존 triple barrier(+3%/-2%/10일) 유지 vs. 뉴스 이벤트별 단기(1~3일) 라벨 병행
- [ ] 슬라이딩 윈도우 재설계: 뉴스 이벤트 중심 윈도우 vs. 고정 60일 윈도우
- [ ] 학습/검증/테스트 분할: 시간순 70/15/15 유지, 시장 레짐별 분리도 고려

---

## 4. 예측 모델 개발 (Model Development)

세 갈래를 **앙상블**로 결합한다.

- [ ] 가격 예측 모델(시계열): LSTM, Temporal Fusion Transformer, PatchTST 비교
- [ ] 텍스트 모델(NLP): KoFinBERT 파인튜닝으로 "뉴스 → 익일 수익률 방향" 분류
- [ ] 멀티모달 융합 모델: 가격 임베딩 + 텍스트 임베딩을 late fusion 또는 cross-attention으로 결합
- [ ] 강화학습 에이전트(선택): PPO/SAC 기반 매매 정책 (상태=피처, 행동=매수/매도/보유, 보상=수익-리스크)
- [ ] 하이퍼파라미터 튜닝: Optuna, Ray Tune
- [ ] 모델 레지스트리: MLflow로 버전 관리
- [ ] 해석가능성: Attention 시각화, SHAP로 뉴스/지표의 기여도 설명

---

## 5. 백테스팅 및 전략 검증 (Backtesting)

모델의 예측력 ≠ 실제 수익성. 현실적 시뮬레이션이 반드시 필요하다.

- [ ] 백테스트 엔진: `backtrader`, `vectorbt` 또는 자체 구현
- [ ] 거래비용 반영: 수수료(증권사별), 증권거래세(0.18%), 슬리피지
- [ ] 거래 제약: 상·하한가, 호가단위, 체결불가 상황, 동시호가
- [ ] 성과 지표: CAGR, Sharpe, Sortino, MDD, Calmar, Win rate, Profit factor
- [ ] 벤치마크 비교: KOSPI 200, 섹터 ETF 대비 초과수익
- [ ] 워크포워드 분석: 롤링 윈도우로 재학습·재평가
- [ ] 시장 레짐별 성과 분해: 상승장/하락장/횡보장 각각 검증
- [ ] 스트레스 테스트: 2020년 3월, 2022년 하락장 등 극단 구간 재현

## 6. 매매 자동화 엔진 (Execution Engine)

국내 주식 자동매매는 브로커 API 연동이 핵심 관문이다.

- [ ] 브로커 API 선택 및 신청 (한국투자증권 KIS Open API 권장 — REST+Websocket, OAuth 기반)
- [ ] 모의투자 환경부터 연동 후 실전 계좌 전환 (실전/모의 각각 앱 키 발급)
- [ ] 계좌·잔고·주문 기능 래퍼: 매수·매도·정정·취소·잔고조회·체결내역
- [ ] 주문 실행 로직: 모델 신호 → 수량 산정 → 호가 전략(시장가/지정가) → 주문 제출 → 체결 확인
- [ ] 포지션 관리: 평균단가, 미실현손익, 보유기간 추적
- [ ] 리스크 관리 모듈: 종목당 최대비중, 포트폴리오 MDD 한도, 일일 손실한도, kill switch
- [ ] 거래 원장(trade ledger): 모든 신호·주문·체결을 DB에 로깅

---

## 7. 실시간 운영 시스템 (Real-time Runtime)

백테스트와 실전의 가장 큰 차이는 "실시간"이다.

- [ ] 스케줄 분리: 장전(09:00 전 뉴스 수집·예측) / 장중(실시간 모니터링·집행) / 장후(결과 집계·재학습 준비)
- [ ] 실시간 시세 스트리밍: KIS Websocket으로 호가·체결 구독
- [ ] 실시간 뉴스 피드: RSS 폴링 or 웹훅, 속보 감지 시 즉시 추론 트리거
- [ ] 모델 서빙: FastAPI/TorchServe, 추론 지연(latency) 수백 ms 이내 목표
- [ ] 시그널 큐: Redis/Kafka로 예측 → 집행 연결
- [ ] 장애 복구: 네트워크 단절 시 재연결, 미체결 주문 상태 동기화
- [ ] 거래 시간·휴장일 캘린더 관리

## 8. 모니터링·대시보드 (Observability)

하루만 지나도 무슨 일이 있었는지 알기 어렵다. 초기부터 설계한다.

- [ ] 실시간 대시보드: 현재 포지션, 오늘 손익, 시그널 로그 (Streamlit, Grafana 또는 React+Chart.js)
- [ ] 모델 드리프트 감지: 예측분포·피처분포 변화 모니터링 (Evidently AI 등)
- [ ] 알림 시스템: 텔레그램/Slack 봇으로 체결·오류·리스크 한도 도달 알림
- [ ] 구조화 로깅: JSON 로그 + 중앙 수집(ELK, Loki)
- [ ] 일일 리포트 자동 생성: 수익률, 거래 요약, 모델 정확도

## 9. 인프라·운영·컴플라이언스 (MLOps & Compliance)

- [ ] 코드 레포 구조: `data/`, `features/`, `models/`, `execution/`, `monitoring/` 모듈 분리 (기존 `stock_data_pipeline.py`는 `data/` 하위로 재배치)
- [ ] 버전관리: Git + DVC(데이터) + MLflow(모델)
- [ ] CI/CD: 유닛 테스트, 통합 테스트(모의투자 연결), 배포 파이프라인
- [ ] 시크릿 관리: API 키는 환경변수 or HashiCorp Vault (Git에 절대 커밋 금지)
- [ ] 서버 배포: 클라우드(AWS/GCP) 또는 국내 VPS, 장중 상시 가동
- [ ] 백업·재현성: 모델·데이터·코드 세 축 모두 버전 태깅
- [ ] 컴플라이언스 확인: 증권사 약관·한국거래소 알고리즘 거래 규정 검토 (자동매매 시 알고리즘 계좌 등록 가능)
- [ ] 자본시장법·개인정보보호법상 주의: 뉴스·SNS 수집 시 저작권·크롤링 정책 준수

## 추천 진행 순서

현재 1번(가격 데이터) 절반이 완성된 상태. 실전 로드맵은 다음과 같다:

1. **Phase 1 (기반)**: 뉴스 수집·전처리 파이프라인(2, 3번) → 기존 가격 피처와 정합
2. **Phase 2 (모델)**: 멀티모달 모델 학습·백테스트(4, 5번)
3. **Phase 3 (집행)**: KIS 모의투자 연동·자동매매(6, 7번) — 반드시 모의로 충분히 검증
4. **Phase 4 (운영)**: 모니터링·실전 배포(8, 9번)
